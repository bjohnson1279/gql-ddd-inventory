## 2024-06-02 - Added ARIA Label and Title to Icon-only Remove Button
**Learning:** Screen readers announce the `&times;` (×) HTML entity as "times," which can be misleading in the context of a remove/delete button for an onboarding variant item.
**Action:** When using `&times;` or icon-only buttons for actions like removing or closing items, ensure to add `aria-label="Remove item"` for screen reader compatibility, and `title="Remove item"` for a native browser tooltip that helps sighted users understand the action.## 2024-06-12 - Ensure Interactive DIVs have Roles
**Learning:** When adding keyboard navigation (`tabIndex={0}` and `onKeyDown`) to non-semantic interactive elements like `<div>` that function as buttons or tabs, it is critical to also provide `role="button"` or `role="tab"`. Without the ARIA role, screen readers will focus the element but won't announce it as an actionable item to the user.
**Action:** Always pair `tabIndex={0}` with an appropriate ARIA role when making `div` or `span` elements interactive, or ideally refactor them to use native `<button>` tags when possible.
